KBAPIConnectionDelegate Protocol Reference

Conforms to NSObject
Declared in KBAPIConnectionDelegate.h

Overview

KBAPIConnection instances' delegates must implement this protocol to receive connection-related events. Note, that all methods from “Handling responses” section are optional, but the delegate should implement one of the methods to actually use the KBAPIConnection instances.

Delegate’s methods are checked in the following order:

  1. If the connection has expected response class, then the apiConnection:didReceiveResponse: is tried.
  2. One of apiConnection:didReceiveJSON: and apiConnection:didReceiveXML: is tried, depending on the response type.
  3. apiConnection:didReceiveData: is tried.

Parameters

connection

Connection which received error.

error

Discussion

This method is called not only on network errors, but also on malformed JSON or XML response or when object cannot be constructed from given source.
